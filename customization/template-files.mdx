## header.php

The header template contains the opening HTML structure and loads WordPress's head section:

```php header.php
<!DOCTYPE html>
<html <?php language_attributes(); ?>>
<head>
    <meta charset="<?php bloginfo('charset'); ?>">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <?php wp_head(); ?>
</head>

<body <?php body_class(); ?>>

<?php
```

<Steps>
  <Step title="HTML5 Doctype">
    Declares modern HTML5 document structure
  </Step>
  <Step title="language_attributes()">
    Outputs the site's language and text direction attributes
  </Step>
  <Step title="bloginfo('charset')">
    Outputs the character encoding (usually UTF-8)
  </Step>
  <Step title="wp_head()">
    Critical hook where WordPress and plugins inject CSS, JS, and meta tags
  </Step>
  <Step title="body_class()">
    Adds contextual CSS classes to the body element
  </Step>
</Steps>

<Warning>
**Never remove `wp_head()`** - Plugins and WordPress core rely on this hook. Removing it will break functionality.
</Warning>

### Using header.php

Load the header in any template file:

```php
<?php get_header(); ?>
```

### Customizing the Header

To add a navigation bar or site logo, you can:

1. **Add directly to header.php:**

```php header.php
<body <?php body_class(); ?>>

<?php get_template_part('template-parts/navbar'); ?>
```

2. **Use the `wp_body_open` hook** (recommended for flexibility):

```php inc/setup.php
function tohka_add_navbar() {
    get_template_part('template-parts/navbar');
}
add_action('wp_body_open', 'tohka_add_navbar');
```

## footer.php

The footer template contains the closing HTML and WordPress footer hook:

```php footer.php

<?php wp_footer(); ?>
</body>
</html>
```

<Warning>
**Never remove `wp_footer()`** - This hook is where JavaScript files are typically loaded, including WordPress admin bar scripts.
</Warning>

### Using footer.php

Load the footer in any template file:

```php
<?php get_footer(); ?>
```

### Customizing the Footer

Add footer content before `wp_footer()`:

```php footer.php
<footer class="site-footer">
    <div class="container">
        <p>&copy; <?php echo date('Y'); ?> <?php bloginfo('name'); ?></p>
    </div>
</footer>

<?php wp_footer(); ?>
</body>
</html>
```

## index.php

The index template is WordPress's ultimate fallback—it renders when no more specific template exists:

```php index.php
<?php get_header(); ?>

<?php
while ( have_posts() ) :
    the_post();
    the_content();
endwhile;
?>

<?php get_footer(); ?>
```

### The WordPress Loop

The code above demonstrates the WordPress Loop—the standard way to display posts:

<Steps>
  <Step title="have_posts()">
    Checks if there are posts to display
  </Step>
  <Step title="the_post()">
    Sets up post data for template tags
  </Step>
  <Step title="the_content()">
    Outputs the post content
  </Step>
</Steps>

### Enhancing index.php

A more complete version might include:

```php index.php
<?php get_header(); ?>

<main class="site-main">
    <?php
    if ( have_posts() ) :
        while ( have_posts() ) :
            the_post();
            ?>
            <article id="post-<?php the_ID(); ?>" <?php post_class(); ?>>
                <h2>
                    <a href="<?php the_permalink(); ?>">
                        <?php the_title(); ?>
                    </a>
                </h2>
                
                <?php if ( has_post_thumbnail() ) : ?>
                    <div class="post-thumbnail">
                        <?php the_post_thumbnail('large'); ?>
                    </div>
                <?php endif; ?>
                
                <div class="entry-content">
                    <?php the_excerpt(); ?>
                </div>
                
                <a href="<?php the_permalink(); ?>" class="read-more">
                    Read More
                </a>
            </article>
            <?php
        endwhile;
        
        the_posts_pagination();
    else :
        ?>
        <p>No posts found.</p>
        <?php
    endif;
    ?>
</main>

<?php get_footer(); ?>
```

## Template Parts

Tohka Theme uses template parts for reusable components:

### template-parts/navbar.php

```php template-parts/navbar.php
<nav class="navbar">
    <div class="container">

        <div class="logo">
            <?php the_custom_logo(); ?>
        </div>

        <?php
        wp_nav_menu(array(
            'theme_location' => 'primary',
            'menu_class'     => 'menu'
        ));
        ?>

    </div>
</nav>
```

### Loading Template Parts

Use `get_template_part()` to include template parts:

```php
<?php get_template_part('template-parts/navbar'); ?>
```

<Tip>
You can pass variables to template parts using `set_query_var()` before loading them.
</Tip>

## WordPress Template Hierarchy

WordPress looks for templates in a specific order. You can create these files to customize different content types:

<Accordion title="Single Posts">
1. `single-{post-type}-{slug}.php`
2. `single-{post-type}.php`
3. `single.php`
4. `singular.php`
5. `index.php`
</Accordion>

<Accordion title="Pages">
1. `page-{slug}.php`
2. `page-{id}.php`
3. `page.php`
4. `singular.php`
5. `index.php`
</Accordion>

<Accordion title="Archives">
1. `archive-{post-type}.php`
2. `archive.php`
3. `index.php`
</Accordion>

<Accordion title="Category Archives">
1. `category-{slug}.php`
2. `category-{id}.php`
3. `category.php`
4. `archive.php`
5. `index.php`
</Accordion>

<Accordion title="Search Results">
1. `search.php`
2. `index.php`
</Accordion>

<Accordion title="404 Error">
1. `404.php`
2. `index.php`
</Accordion>

## Creating Custom Templates

### Page Templates

Create custom page templates with this header comment:

```php template-full-width.php
<?php
/**
 * Template Name: Full Width
 * Description: Page template without sidebar
 */

get_header();
?>

<div class="full-width-content">
    <?php
    while ( have_posts() ) :
        the_post();
        the_content();
    endwhile;
    ?>
</div>

<?php get_footer(); ?>
```

Users can select this template from the Page Attributes meta box in the WordPress editor.

### Single Post Templates

Create `single.php` for individual post display:

```php single.php
<?php get_header(); ?>

<main class="site-main">
    <?php
    while ( have_posts() ) :
        the_post();
        ?>
        <article id="post-<?php the_ID(); ?>" <?php post_class(); ?>>
            <header class="entry-header">
                <h1><?php the_title(); ?></h1>
                
                <div class="entry-meta">
                    <span class="author">
                        By <?php the_author(); ?>
                    </span>
                    <span class="date">
                        <?php echo get_the_date(); ?>
                    </span>
                </div>
            </header>
            
            <?php if ( has_post_thumbnail() ) : ?>
                <div class="featured-image">
                    <?php the_post_thumbnail('large'); ?>
                </div>
            <?php endif; ?>
            
            <div class="entry-content">
                <?php the_content(); ?>
            </div>
        </article>
        <?php
    endwhile;
    ?>
</main>

<?php get_footer(); ?>
```

### Archive Templates

Create `archive.php` for category, tag, and date archives:

```php archive.php
<?php get_header(); ?>

<main class="site-main">
    <header class="archive-header">
        <h1><?php the_archive_title(); ?></h1>
        <?php the_archive_description('<div class="archive-description">', '</div>'); ?>
    </header>
    
    <?php
    if ( have_posts() ) :
        while ( have_posts() ) :
            the_post();
            get_template_part('template-parts/content', 'excerpt');
        endwhile;
        
        the_posts_pagination();
    endif;
    ?>
</main>

<?php get_footer(); ?>
```

## Template Tags

Common WordPress template tags you'll use:

| Template Tag | Purpose |
|--------------|--------|
| `the_title()` | Outputs post/page title |
| `the_content()` | Outputs full post content |
| `the_excerpt()` | Outputs post excerpt |
| `the_permalink()` | Outputs post URL |
| `the_post_thumbnail()` | Outputs featured image |
| `the_author()` | Outputs author name |
| `the_date()` | Outputs publish date |
| `body_class()` | Outputs contextual body classes |
| `post_class()` | Outputs contextual post classes |

<Note>
Most template tags must be used inside the WordPress Loop after calling `the_post()`.
</Note>

## Best Practices

<Steps>
  <Step title="Always Include wp_head() and wp_footer()">
    These hooks are essential for WordPress and plugin functionality
  </Step>
  <Step title="Use Template Parts for Reusability">
    Extract repeated code into `template-parts/` for easier maintenance
  </Step>
  <Step title="Follow the Template Hierarchy">
    Create specific templates only when needed—let WordPress fall back to index.php otherwise
  </Step>
  <Step title="Use Template Tags Correctly">
    Only use template tags like `the_title()` inside the Loop
  </Step>
  <Step title="Add Semantic HTML">
    Use proper HTML5 elements like `<article>`, `<header>`, `<main>`, and `<footer>`
  </Step>
</Steps>
