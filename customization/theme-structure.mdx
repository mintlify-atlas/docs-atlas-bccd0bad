Tohka Theme follows a clean, modular architecture that separates concerns and makes customization straightforward. Understanding the directory structure is essential for effective theme development.

## Directory Overview

Here's the complete file structure of Tohka Theme:

```
tohka-theme/
├── assets/
│   ├── css/
│   │   └── ani.css
│   └── js/
│       └── main.js
├── inc/
│   ├── setup.php
│   ├── assets.php
│   └── sidebar.php
├── template-parts/
│   └── navbar.php
├── functions.php
├── style.css
├── header.php
├── footer.php
├── index.php
└── screenshot.png
```

## Core Theme Files

### functions.php

The main entry point that loads all modular components. This file deliberately keeps logic minimal:

```php
<?php
require get_template_directory() . '/inc/setup.php';
require get_template_directory() . '/inc/assets.php';
require get_template_directory() . '/inc/sidebar.php';
```

<Note>
**Design Philosophy:** All business logic lives in the `/inc` directory. Never add custom functions directly to `functions.php`—create a new module in `/inc` instead.
</Note>

### style.css

Contains theme metadata required by WordPress:

```css
/*
Theme Name: Tohka Theme
Theme URI: 
Author: Beyond the Horizon Team
Author URI: 
Description: Tohka es un tema moderno, ligero y optimizado para rendimiento
Version: 1.0.0
Requires at least: 6.0
Tested up to: 6.5
Requires PHP: 7.4
License: GPL v2 or later
Text Domain: tohka.com
*/
```

<Warning>
Don't delete `style.css`—WordPress requires it to recognize your theme. The actual stylesheets are loaded from `assets/css/`.
</Warning>

### inc/setup.php

Configures WordPress theme features:

```php inc/setup.php
function tokha_setup() {
    add_theme_support('title-tag');
    add_theme_support('post-thumbnails');
    add_theme_support('html5', array(
        'search-form',
        'comment-form',
        'comment-list',
        'gallery',
        'caption'
    ));
    add_theme_support('align-wide');
    
    register_nav_menus(array(
        'primary' => __('Menú Principal', 'tokha'),
    ));
}

add_action('after_setup_theme', 'tokha_setup');
```

### inc/assets.php

Manages all CSS and JavaScript enqueuing:

```php inc/assets.php
function tokha_assets() {
    $css_files = array('ani');
    
    foreach ($css_files as $file) {
        wp_enqueue_style(
            'tohka-' . $file,
            get_template_directory_uri() . "/assets/css/{$file}.css",
            array(),
            '1.0',
            'all'
        );
    }
    
    wp_enqueue_script(
        'tokha-main',
        get_template_directory_uri() . '/assets/js/main.js',
        array(),
        filemtime(get_template_directory() . '/assets/js/main.js'),
        true
    );
}

add_action('wp_enqueue_scripts', 'tokha_assets');
```

### inc/sidebar.php

Registers widget areas:

```php inc/sidebar.php
function tohka_widgets_init() {
    register_sidebar(array(
        'name'          => 'Sidebar Principal',
        'id'            => 'sidebar-1',
        'description'   => 'Widgets del sidebar principal',
        'before_widget' => '<div class="widget">',
        'after_widget'  => '</div>',
        'before_title'  => '<h3 class="widget-title">',
        'after_title'   => '</h3>',
    ));
}

add_action('widgets_init', 'tohka_widgets_init');
```

## The /assets Directory

Stores all static files:

<Steps>
  <Step title="CSS Files">
    Located in `assets/css/`. Add new stylesheets here and reference them in `inc/assets.php`
  </Step>
  <Step title="JavaScript Files">
    Located in `assets/js/`. The main script is `main.js`, loaded in the footer for performance
  </Step>
</Steps>

<Tip>
Use `filemtime()` for JavaScript versioning to automatically bust cache when files change.
</Tip>

## The /template-parts Directory

Contains reusable template components:

```php template-parts/navbar.php
<nav class="navbar">
    <div class="container">
        <div class="logo">
            <?php the_custom_logo(); ?>
        </div>

        <?php
        wp_nav_menu(array(
            'theme_location' => 'primary',
            'menu_class'     => 'menu'
        ));
        ?>
    </div>
</nav>
```

Load template parts using:

```php
<?php get_template_part('template-parts/navbar'); ?>
```

## Where to Add Custom Code

<Accordion title="Adding New Features">
Create a new PHP file in `/inc` (e.g., `inc/custom-post-types.php`) and require it in `functions.php`:

```php functions.php
require get_template_directory() . '/inc/custom-post-types.php';
```
</Accordion>

<Accordion title="Adding Custom CSS">
Create a new CSS file in `assets/css/` and add it to the array in `inc/assets.php`:

```php inc/assets.php
$css_files = array('ani', 'custom');
```
</Accordion>

<Accordion title="Adding Custom JavaScript">
Enqueue it in `inc/assets.php` using `wp_enqueue_script()`:

```php inc/assets.php
wp_enqueue_script(
    'tohka-slider',
    get_template_directory_uri() . '/assets/js/slider.js',
    array('jquery'),
    filemtime(get_template_directory() . '/assets/js/slider.js'),
    true
);
```
</Accordion>

<Accordion title="Adding Template Parts">
Create new PHP files in `template-parts/` and load them with `get_template_part()`:

```php
<?php get_template_part('template-parts/hero'); ?>
```
</Accordion>

## Organization Philosophy

Tohka Theme follows these principles:

1. **Separation of Concerns** - Each file has a single, clear responsibility
2. **Modularity** - Features are isolated in `/inc` modules
3. **Minimal Core** - `functions.php` only loads modules, never contains logic
4. **Clear Naming** - File names describe their purpose
5. **WordPress Standards** - Follows official WordPress coding standards

<Note>
This architecture makes the theme easy to maintain, debug, and extend without creating conflicts.
</Note>
